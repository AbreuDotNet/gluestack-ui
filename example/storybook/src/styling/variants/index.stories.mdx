# Variants

Creating variants of components in a UI component library is crucial for maintaining a consistent user experience across `web`, `iOS`, and `android` platforms. We can create different variants of components using **gluestack-style**.

To update components styles, we need to create a new config which extends the default [`config`](https://github.com/gluestack/gluestack-ui/blob/main/packages/config/src/gluestack-ui.config.ts) file imported from [**`@gluestack-ui/config`**](https://github.com/gluestack/gluestack-ui/blob/main/packages/config) and pass it inside the `GluetackUIProvider`, then we can style the components additionally.

To create a new variant or extend styling of a Button Component, just create a config file `gluestack-ui.config.ts` in your project root. Now import import the `createConfig` function from **`@gluestack-style/react`** and pass the `config` object to the function. Inside the components object define the component name which is imported from **`@gluestack-ui/themed`** library, and then all the styles that needs to be applied on the component.

```jsx
// gluestack-ui.config.ts
import { config as gluestackUIConfig } from "@gluestack-ui/config";
import { createConfig, createComponents } from "@gluestack-style/react";

export const customConfig = createConfig({
  // ... other configuration
  components: createComponents({
    Button: {
      // write same component name which is passed in styled
      theme: {
        bg: "$primary500",
        variants: {
          variant: {
            ghost: {
              backgroundColor: "$red500",
              _text: {
                color: "white",
              },
            },
          },
        },
      },
      componentConfig: {
        descendantStyle: ["_text"],
      },
    },
  }),
  // ... other configuration
});

// Merge with default configuration provided by @gluestack-ui/config
const export config = deepMerge(gluestackUIConfig, customConfig);

type Config = typeof config;
type Components = typeof components;

declare module '@gluestack-ui/themed' {
  interface UIConfig extends Config {}
  interface UIComponents extends Components {}
}

```

Now you can pass this config to `GluestackUIProvider`.

<AppProvider>
  <Center>
    <Button
      sx={{
        backgroundColor: '$red500',
        _text: {
          color: 'white',
        },
      }}
    >
      <ButtonText>Hello world</ButtonText>
    </Button>
  </Center>
</AppProvider>
<br />

```jsx
//App.tsx
import {config} from './gluestack-ui.config.ts';
import {GluestackUIProvider, Box, Button, ButtonText} from './gluestack-ui.config.ts';

function App(){
    return (
     <GluestackUIProvider config={config}>
        <Box justifyContent="center" alignItems="center" height="100%">
          {/* Button with default theme */}
          <Button>
            <ButtonText>Hello world</ButtonText>
          </Button>
        </Box>
      </GluestackUIProvider>
);
```

You can refer to [this](/ui/docs/theme-configuration/customizing-theme) link for detailed information related to customizing theme of components.
